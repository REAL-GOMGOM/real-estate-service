## 3. 🔴 이번에 겪은 사건·교훈 (재발 방지)

1. **Cache Components 프리렌더 규칙** — 페이지/레이아웃에서 **비캐시 IO(직접 DB 등) 금지**. 반드시 `'use cache' + cacheLife` 경계로 감싸기 (레포 표준 패턴: `lib/subscription-api.ts:354`). try/catch는 소용없음 — Next가 IO 자체를 검출한다.
   - 추가 (07-11): **`export const dynamic` 등 세그먼트 설정도 cacheComponents 와 비호환** — 빌드가 즉사한다 (실제 발생). 라우트를 요청 시점으로 빼려면 핸들러 안에서 **`await connection()`** (`next/server`) 사용.
2. **게이트는 && 체인으로** — vitest/build 와 git 명령을 따로 붙여넣으면 빌드 실패에도 push 되는 사고가 **두 번** 발생 (52090cc·8711109→핫픽스). 이후로는 `npx vitest run && npm run build && git add … && git commit … && git push` 한 줄 체인 사용 — 실패 시 자동 중단.
3. **내집봇 토큰 노출·폴링 충돌 사건** — BotFather 옛 대화의 토큰(8636…)을 새 봇 토큰으로 오인해 Hermes에 투입 → 프로덕션 내집봇과 getUpdates 충돌 + 채팅 노출. **로테이션 절차**(Revoke→`~/.openclaw/.env` MYZIP_BOT_TOKEN 교체→kickstart)로 수습. 교훈: 토큰은 캡처 포함 어디에도 올리지 않기, `8636…=내집봇`, 충돌은 "수신(폴링)"만 발생하고 발송(sendMessage)은 무관.
4. **봇 pytest 인터프리터** — `python3.14 -m pytest`. 3.9로 돌리면 `fetch_realestate.py`의 `str | None`에서 collection error (코드 문제 아님).
5. zsh: `setopt interactive_comments` 적용됨(주석 붙여넣기 OK), 커밋 메시지 `!` 금지 유지.
